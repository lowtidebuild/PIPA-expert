# PIPA Expert 사용 가이드

[English](../en/HOW-TO-USE.md) | [한국어](./HOW-TO-USE.md)

> 이 가이드는 **비개발자**를 위해 작성되었습니다. Python, Git, API를 몰라도 괜찮습니다. 질문을 타이핑할 수 있으면 이 도구를 사용할 수 있습니다.

## 필요한 것 (최초 1회 설정)

| 항목 | 이유 | 설치 방법 |
|------|------|---------|
| **Claude Code** | 에이전트를 실행하는 앱 | [시작하기](https://docs.anthropic.com/en/docs/claude-code/overview) — CLI, 데스크톱 앱, VS Code 확장 |
| **이 저장소** | 법령 Knowledge Base + 에이전트 포함 | GitHub에서 다운로드 (아래 참조) |

이게 전부입니다. 데이터베이스, 서버, API 키 설정이 필요 없습니다 (법령 데이터를 직접 갱신하려는 경우 제외).

## 저장소 다운로드

이 프로젝트의 로컬 복사본이 컴퓨터에 필요합니다.

### Git이 설치되어 있는 경우

터미널을 열고 실행:

```bash
git clone <repository-url>
```

모든 법령 데이터와 에이전트 파일이 포함된 `PIPA-expert` 폴더가 생성됩니다. 현재 저장소의 호스팅 페이지에서 실제 URL을 복사해 사용하세요.

### Git이 없는 경우

1. 현재 저장소의 메인 호스팅 페이지에 접속
2. 초록색 **"Code"** 버튼 클릭
3. **"Download ZIP"** 클릭
4. 다운로드한 파일을 원하는 폴더에 압축 해제 (예: `Documents/PIPA-expert`)

---

## 에이전트 시작하기

### 방법 A: 데스크톱 앱 / VS Code

1. Claude Code 실행
2. 다운로드한 `PIPA-expert` 폴더 열기
3. `/agents/pipa-agent`를 입력하여 PIPA 전문 에이전트 활성화

### 방법 B: 터미널 (CLI)

```bash
cd PIPA-expert
claude
```

이후 `/agents/pipa-agent`를 입력하여 에이전트를 활성화합니다.

## 질문하기

자연어로 질문하면 됩니다. 에이전트는 **한국어와 영어** 모두 지원하며, 사용하는 언어로 응답합니다.

### 단순 조회

> "개인정보보호법 제15조 보여줘"

> "정보통신망법 제22조 내용은?"

> "개인정보 처리 통합 안내서 가이드라인 찾아줘"

### 법률 분석

> "맞춤형 광고를 위해 행태정보를 수집할 때 동의 구조는 어떻게 설계해야 해?"

> "위탁과 제3자 제공의 구분 기준과 실무적 차이는?"

> "정보통신망법과 개인정보보호법의 동의 규정 차이점은?"

### 교차참조

> "제15조의 시행령 위임 조문은?"

> "가명정보 처리 관련 가이드라인 전부 보여줘"

> "제3자 제공에 관한 조문과 관련 시행령, 가이드라인을 교차 확인해 줘"

### 비교 분석

> "개인정보보호법과 신용정보법의 가명처리 규정 비교해 줘"

> "위치정보법과 개인정보보호법의 동의 요건 차이는?"

## 법률의견서 요청하기 (DOCX)

채팅 답변이 아닌 정식 문서가 필요할 때 — 법률의견서를 요청하세요:

> "맞춤형 광고 동의 구조 재설계 방안에 대한 법률의견서 작성해줘"

> "제3자 제공 관련 법률의견서를 DOCX로 만들어줘"

> "AI 학습용 데이터 수집의 적법성에 대한 의견서 작성해줘"

에이전트가 수행하는 작업:
1. 전체 Knowledge Base 검색 (550개 검색 가능한 법조문 파일 + 46건 가이드라인)
2. 검증된 인용이 포함된 구조화된 의견서 초안 작성
3. 팩트체커로 모든 법적 인용 검증
4. 전문적인 DOCX 파일을 `${PIPA_OUTPUT_DIR:-output/opinions/}`에 저장

쟁점이 `제7조의2` 같은 가지조문에 걸려 있다면, 에이전트에게 `law.go.kr` 재검증까지 함께 요청하는 것이 좋습니다. 현재 로컬 법령 인덱스는 일부 법령에서 기본 조문 번호 기준으로 평탄화되어 있습니다.

## 결과물 이해하기

### 인용 태그

에이전트 출력의 모든 법적 인용에는 신뢰도 태그가 붙습니다:

| 태그 | 의미 | 신뢰할 수 있나? |
|------|------|---------------|
| `[VERIFIED] [Grade A]` | KB의 법령 원문 또는 PIPC 가이드라인과 정확히 매칭 | 높은 신뢰도 — 단, 중요한 포인트는 항상 재확인 |
| `[VERIFIED] [Grade B]` | KB의 개보위 처분례 또는 법원 판례와 매칭 | 양호한 신뢰도 — 권위 있지만 2차 소스 |
| `[UNVERIFIED]` | 웹 검색으로 발견, 로컬 KB에 없음 | 의존 전 독립적으로 검증 필요 |
| `[INSUFFICIENT]` | 에이전트가 충분한 근거를 찾지 못함 | 에이전트가 솔직한 것 — 추측 말고 변호사 상담 |
| `[CONTRADICTED]` | 소스마다 다른 내용 | 양쪽 모두 제시됨 — 판단은 사용자 몫 |

### 소스 등급

| 등급 | 포함 내용 | 단독 근거로 사용 가능? |
|------|---------|---------------------|
| **A** | 법령 원문, 시행령, 개보위 공식 가이드라인 | 가능 |
| **B** | 개보위 처분례, 법원 판례 | 가능 (Grade A 교차검증 권장) |
| **C** | 로펌 분석, 학술 논문 | 불가 — 참고용으로만 |
| **D** | 뉴스 기사, AI 요약 | 시스템에서 완전 제외 |

## 나만의 소스 추가하기

개보위 처분례, 로펌 뉴스레터, 학술 논문 등을 에이전트에 추가하고 싶다면?

### Step 1: 파일 넣기

아무 파일(PDF, DOCX, HTML)을 `${PIPA_INBOX_DIR:-library/inbox/}` 폴더에 넣습니다.

### Step 2: 에이전트에 알리기

> "inbox에 파일 넣었어, ingest 해줘"

또는:

> "/ingest"

### Step 3: 완료

에이전트가 자동으로:
- 파일을 Markdown으로 변환
- 신뢰도 자동 분류 (Grade A, B, C)
- 메타데이터 추출 (제목, 날짜, 키워드, 관련 조문)
- 적절한 폴더에 배치
- 검색 인덱스 업데이트

새 소스가 바로 검색 가능하고, 향후 의견서에서 인용될 수 있습니다.

### 추가할 수 있는 자료 (예시)

| 자료 | 처리 결과 |
|------|---------|
| 개보위 공식 가이드라인 PDF | Grade A로 분류, 가이드라인 폴더에 배치 |
| 개보위 처분례 | Grade B로 분류, 처분례 폴더에 배치 |
| 김장 법률사무소 뉴스레터 | Grade C로 분류, 참고자료 폴더에 배치 |
| 언론 기사 (개인정보 유출 보도) | Grade D로 거부 (경고 메시지 표시) |

## 좋은 결과를 위한 팁

### 구체적으로 질문하기

| 이렇게 말고 | 이렇게 |
|-----------|--------|
| "개인정보보호법에 대해 알려줘" | "개인정보보호법 제15조의 적법처리 근거 6가지는?" |
| "이거 합법이야?" | "제3자 제공 시 동의 없이 가명처리된 정보를 제공할 수 있는지 분석해 줘" |
| "데이터 수집 어떻게 해?" | "정보통신망법상 행태정보 수집 시 동의 요건과 개인정보보호법상 요건 비교" |

### 반론도 요청하기

> "이 처리에 정당한 이익을 근거로 삼는 것에 반대하는 논거는?"

에이전트에 adversarial 모드가 내장되어 있어서 예외, 반대 판례, 대안적 해석을 찾지만, 명시적으로 요청하면 더 효과적입니다.

### 팩트체크 리포트 요청하기

중요한 의견서의 경우, 검증 결과를 확인하세요:

> "이 의견서의 팩트체크 리포트 보여줘"

어떤 인용이 검증되었는지, 문제가 있었는지, 전체 신뢰도 점수를 볼 수 있습니다.

## 법령 데이터 갱신

법령 데이터는 [국가법령정보센터 Open API](https://open.law.go.kr)에서 수집합니다. 갱신하려면:

```bash
python3 scripts/fetch-pipa-from-api.py --oc 이메일ID
```

Open Law API 무료 계정이 필요합니다. `--oc` 파라미터는 등록된 이메일 ID입니다. 월 1회 또는 법률 개정 시 갱신을 권장합니다.

---

## 이 도구가 하지 않는 것

- **법률 자문을 제공하지 않습니다.** 법률 소스를 더 빠르고 체계적으로 찾도록 도와주는 리서치 보조 도구입니다. 결과물은 반드시 법률 전문가가 검토해야 합니다.
- **귀하의 구체적인 계약이나 내부 정책을 알지 못합니다** — ingest 시스템으로 추가하지 않는 한.
- **자동으로 업데이트되지 않습니다.** 법령 데이터는 특정 시점의 스냅샷입니다. 주기적으로 갱신 스크립트를 실행하거나 ingest로 새 소스를 추가하세요.
- **현재 한국 개인정보보호 법령만 다룹니다.** EU/GDPR은 별도 저장소 `GDPR-expert`를 참고하세요.

## 도움이 필요할 때

- **뭔가 이상해 보인다면?** 에이전트에게: "제15조 인용을 검증해줘"
- **다른 포맷이 필요하다면?** "마크다운 요약으로 줘" 또는 "DOCX로 만들어줘"
- **에이전트가 막힌 것 같다면?** 질문을 다시 표현하거나 작은 부분으로 나눠서 시도


---

> **기억하세요:** 이것은 자동 조종이 아니라 파워 도구입니다. 법률 리서치를 극적으로 빠르고 체계적으로 만들어주지만, 최종 판단은 항상 자격을 갖춘 사람의 몫입니다.
